使用hexo与github搭建个人免费博客的安装流程与步骤

发表于 2018-04-16 | 分类于 搭建博客

最近研究了一下怎样安装hexo写博客，并搭建至Github上，毕竟现在出去找工作多数公司都要求有
自己的个人博客并在Github上贡献过代码，所以为了赶上需求，我也搭建个个人博客，没事写写开发过程
中的一些心的，也算做个总结，今天就记载并分享一下搭建博客的整个步骤，毕竟自己刚搭建完成，也算
有点经验哈，话不多说，开始：

安装 Node.js

因为Hexo基于Node.js的，那么我们搭建博客网站首先需要安装Node.js环境。 Node.js 是一个基于 
Chrome V8 引擎的 JavaScript 运行环境，可以在非浏览器环境下，解释运行 JS 代码。 
去官网（http://node.org）进行下载最新的nodejs版本，下载后一路默认就行，实在不会就找

安装Git

Git是目前世界上最流行的分布式版本控制系统。使用Git可以帮助我们把本地的网页和文章等内容提交到Gihub上，实现同步。 
下载地址：https://git-scm.com/downloads

查看显示版本号即成功。

安装hexo博客框架

hexo是什么？
官网给出：A fast, simple & powerful blog framework
就是一款快速、简单、强大的博客框架
安装起来也很简单，官网(http://hexo.io)也给出来了安装步骤：

  $ npm install hexo-cli -g (这句话就是安hexo到语句，若出错就在前加sudo，就是以管理员权限执行，输入管理员密码即可)
  $ hexo init blog  ／／初始化博客文件夹
  $ cd blog ／／切换到博客路径
  $ npm install ／／ 安装其他插件
  $ hexo server  ／／运行博客

运行博客点击回车后，若成功就会出现http://localhost:4000地址，在浏览器输入网址，就会显示整个博客的架构

部署到github上

##申请账号
1.若已有github账号的此部跳过
2.没有就去官网(http://github.com)申请一个账号，具体步骤可以百度

##新建一个项目
1.登入github后点击”Start a project”或点击有上角”＋”下拉菜单中的new repository进行项目的创建
2.进入新建项目页面后，项目必须要遵守格式：账户名.github.io，不然接下来会有很多麻烦。并且需要勾选Initialize this repository with a README
3.前后连个账户名一定要相同不然，部署完成后会出现404的错误
4.点击”create repository”后，项目算创建完成，点击右侧settings按钮，点击它，向下拉到GitHub Pages，你会看到那边有个网址，访问它，你将会惊奇的发现该项目已经被部署到网络上，能够通过外网来访问它。
5.配置hexo里的config_yml文件

##发布到github上
输入如下命令：hexo clean && hexo g && hexo d

第一次发布需要验证github账号,之后发布就不用了
现在再次访问链接http://用户名.github.io就进入了刚发布的博客网页，如此hexo个人博客
搭建在github上就算完成了
